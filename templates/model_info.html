📚 Source data

The system uses the BGS Africa Groundwater Atlas (Zimbabwe extract, 1:5,000,000) as its hydrogeological reference. Every GPS point is spatial-joined against this shapefile to derive the geology class and the BGS regional baseline yield.

See Data Sources for full attribution, licence, and the loaded dataset status.

Post-prediction processing

Once the SVM produces its raw class, the system runs three more passes before reporting:

  1. BGS regional baseline cross-check — compare the SVM's binary class to the BGS Africa Groundwater Atlas baseline yield class for the polygon at the GPS point. Disagreement is treated as a sign that more on-the-ground information (e.g. electromagnetic surveys) is needed, not as a hard error.
  2. Total Confidence Score (TCS) — sum of four components out of 10:
    • C1 (3) — data completeness: how many of the 6 predictors were directly observed vs inferred
    • C2 (3) — geology match quality: confidence of the BGS-class to model-class remapping
    • C3 (2) — biophysical indicator convergence: how many supplementary observations support the prediction
    • C4 (2) — BGS baseline alignment: 2 if model and BGS agree, 1 if neither agrees nor disagrees, 0 if they conflict
  3. Land Use Pressure (LUPS) modifier — the surveyor's land-use checklist sums to a single integer. Moderate pressure (LUPS −1 to −2) reduces TCS by 1; severe pressure (LUPS ≤ −3) reduces TCS by 2 and downgrades a High prediction to Low.

If any of these conditions fire, the prediction is added to the Expert Review queue:

  • Adjusted TCS < 5
  • Severe land-use pressure (LUPS ≤ −3)
  • Geology remapped with Low confidence
  • No supplementary indicators support the prediction
  • BGS baseline differs from the model
